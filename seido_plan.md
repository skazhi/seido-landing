# Seido — План развития

## Цель
Построить экосистему, где каждый беговой результат, каждый старт, каждый бегун и каждый организатор будут в одном месте.  
**Статус: Главная соцсеть для бегунов в России**.

## Название
**Seido** (произносится /сей-до/)  
Происхождение: **Скажи** (твой ник) + **Done** (сделано)  
Глагол: **«Засейду»** — добавить результат в систему

> *«Ты где результаты смотришь?» — «В Seido, засейду все забеги там»*

## Принципы
- Делать быстро, тестировать на себе и друзьях.
- Сначала бесплатно и полезно, потом монетизация.
- Один шаг за раз, чёткие критерии перехода.

## Легенда
Ты — соло-разработчик с вайбкодингом, имеешь доступ к беговому комьюнити (200+ человек). Бюджет на старте — только твоё время и немного денег на серверы/домены.

## Условные обозначения
- [ ] — задача не выполнена
- [x] — задача выполнена
- **Промт** → текст, который можно скопировать и отправить ИИ (например, ChatGPT) для генерации кода/текста/идей.
- **Тест** — как проверить, что шаг сделан правильно.
- **Хороший/плохой сценарий** — прогноз по деньгам/пользователям.

## Этап 1: MVP — Telegram-бот + база данных (1–2 недели)

**Цель:** Создать инструмент, который уже даёт ценность комьюнити, даже без сложного интерфейса.

- [ ] **1.1 Проектирование базы данных**
  - Определить сущности: бегун (ФИО, дата рождения, пол, город, ссылки на соцсети), забег (название, дата, дистанция, тип, организатор), результат (время, место, возрастная категория, место в категории).
  - Выбрать СУБД: PostgreSQL (бесплатно, мощно).

  > **Промт:**  
  > «Создай SQL-скрипт для создания таблиц PostgreSQL для сервиса беговой статистики. Нужны таблицы: runners, races, results. Добавь внешние ключи, индексы. Комментарии к полям на русском.»

  **Тест:** скрипт выполняется без ошибок, можно добавить тестовую запись.

- [ ] **1.2 Написать Telegram-бота (Python + python-telegram-bot)**
  - **Функции:**
    - Регистрация пользователя (сохранение ФИО, даты рождения, города)
    - Поиск своих результатов (по ФИО или по ID)
    - Показ последних результатов, мест, рекордов
    - Сравнение с другими (по списку друзей)
  - Подключить базу данных (через SQLAlchemy или psycopg2).

  > **Промт:**  
  > *«Напиши код Telegram-бота на Python (aiogram 3.x) для сбора статистики бегунов. Бот должен: 1) приветствовать, 2) регистрировать пользователя (ФИО, дата рождения, город), 3) по команде /myresults показывать все его результаты из базы, 4) по команде /stats показывать общую статистику. Используй PostgreSQL. Разбей код на файлы: main.py, db.py, handlers.py. Добавь обработку ошибок.»*

  **Тест:** бот отвечает на команды, данные сохраняются в БД.

- [ ] **1.3 Ручное наполнение базы (первые протоколы)**
  - Собрать протоколы 5–10 популярных московских забегов (например, «Пятерка», Московский марафон, Кросс Нации и т.п.) в формате PDF/Excel.
  - Вбить данные вручную (или с помощью скрипта-парсера, если просто).

  > **Промт (для парсинга):**  
  > «Напиши Python-скрипт для парсинга результатов забега из PDF-файла в формате таблицы. Используй библиотеку tabula-py или pdfplumber. На выходе должен быть CSV с колонками: место, имя, год рождения, город, время. Учти, что структура PDF может отличаться — добавь возможность ручной корректировки.»

  **Тест:** в таблице results появились записи для конкретных бегунов.

- [ ] **1.4 Запуск бота для своего комьюнити**
  - Пригласить 200+ знакомых бегунов протестировать.
  - Собрать обратную связь.

  > **Промт:**  
  > «Напиши текст приглашения в Telegram-чате для тестирования бота Seido. Упомяни, что они смогут увидеть все свои результаты в одном месте, даже со старых забегов. Добавь ссылку на бота и просьбу написать в личку о найденных багах.»

  **Тест:** 30–50 человек начали пользоваться ботом, есть первые запросы на добавление новых забегов.

### Финансы этапа
| Расходы | Доходы |
|---------|--------|
| Сервер (VPS) — 500–1000 руб/мес | 0 |

## Этап 3: Веб-интерфейс и календарь (1–2 месяца)

**Цель:** Сделать платформу доступной не только через бот, добавить визуализацию и календарь.

- [ ] **3.1 Разработать веб-версию (React/Vue)**
  - Авторизация через Telegram или email.
  - Профиль бегуна с графиками результатов.
  - Календарь забегов с фильтрами.

  > **Промт:**  
  > «Создай React-приложение (Create React App) для SayDo. Главная страница — календарь забегов. Страница профиля — таблица личных рекордов, график изменения темпа по годам. Используй компоненты Material-UI. Добавь роутинг. Для данных используй API, которое будет отдавать FastAPI.»

  **Тест:** веб-версия открывается, данные подтягиваются из бэкенда.

- [ ] **3.2 Написать API (FastAPI) для взаимодействия с БД**
  - Эндпоинты: получение профиля, списка забегов, результатов, добавление результата (для организаторов).

  > **Промт:**  
  > «Напиши FastAPI-приложение с эндпоинтами: GET /runners/{id} — данные бегуна; GET /races — список забегов с пагинацией; POST /results — добавление результата (только для авторизованных организаторов). Используй Pydantic для валидации. Подключи CORS для фронтенда.»

  **Тест:** запросы к API возвращают корректные JSON.

- [ ] **3.3 Запуск календаря забегов**

  - Показывать, кто из друзей планирует участвовать.

  > **Промт:**  
  > «Добавь в API эндпоинты для подписки на забег: POST /races/{id}/subscribe. В профиле пользователя показывать список предстоящих забегов. На фронтенде сделать кнопку "Буду участвовать".»

  - Подключить платежи (ЮKassa или Тинькофф Касса).

  > **Промт:**

  **Тест:** пользователь может оплатить подписку и получить Pro-статус.

## Этап 5: Масштабирование и мобильное приложение (6–12 месяцев)

**Цель:** Выйти на лидерские позиции, автоматизировать всё, расширяться на другие города и виды спорта.

- [ ] **5.1 Мобильное приложение (React Native / Flutter)**
  - Основной функционал сайта + push-уведомления о стартах, новых результатах.

  > **Промт:**  
  > «Создай React Native приложение для SayDo. Скопируй основные экраны из веб-версии. Добавь push-уведомления через Firebase. Настройку сборку под Android и iOS.»

  **Тест:** приложение опубликовано в сторах (хотя бы Android).

- [ ] **5.2 Расширение на другие регионы (Санкт-Петербург, Казань, Екатеринбург и т.д.)**
  - Нанять контент-менеджера для поиска забегов и организаторов в регионах.

  > **Промт:**  
  > «Напиши вакансию контент-менеджера для SayDo. Обязанности: поиск региональных забегов, связь с организаторами, проверка протоколов. Требования: знание беговой тусовки, умение работать с Excel. Зарплата — оклад + бонус за подключенных организаторов.»

  **Тест:** база забегов расширяется, появляются организаторы из новых городов.

- [ ] **5.3 Выход в смежные виды спорта (лыжи, велоспорт, плавание)**
  - Адаптировать платформу под новые типы активностей.
  - Привлечь соответствующие сообщества.

  > **Промт:**  
  > «Проведи анализ: какие виды спорта близки к бегу по структуре соревнований? Предложи стратегию адаптации SayDo под лыжные гонки и триатлон. Напиши список первых мероприятий для пилота.»

- [ ] **5.4 Выход на международный рынок (ближнее зарубежье, затем Европа)**

## Этап ∞: Признание

**Цель:** Достичь статуса «первая ассоциация» для бегунов в России.

### Критерии достижения
- Более 50% активных бегунов России имеют профиль в SayDo.
- Организаторы по умолчанию выгружают результаты через SayDo.
- Слово «saydo» стало нарицательным: «Засейду свои результаты», «Зарегайся на сейду».

### Стратегия удержания лидерства
- Постоянное улучшение UX.
- Внедрение AI-тренеров, прогнозов, социальных функций.
- Партнёрство с крупными брендами (Nike, Adidas) и федерациями.
